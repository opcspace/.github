## OPCspace 是什么

OPCspace 是一个面向 **AI 一人公司与小型组织** 的创业决策与实践社区。

这里的 OPC 不只是“一人开公司”，而是一种新的组织方法：把个人判断、专业知识、业务流程和 AI Agent 组合成可以持续运行、不断复用的组织能力。

我们同时建设三类相互连接的公共资产：

| DECISION INFRASTRUCTURE | OPEN WORKFLOWS | PRACTICE COMMUNITY |
|---|---|---|
| **城市与政策决策基础设施** | **开源工作流与实践知识库** | **全国 OPC 实践社区** |
| 用城市、政策、园区和产业研究帮助创业者做可核查的决策。 | 把创业、增长、合规、技术栈和工具实践封装成可执行资产。 | 连接实践者、城市节点、工具伙伴与服务机构，沉淀真实案例。 |
| [访问中文官网](https://opcspace.com.cn/) | [浏览旗舰项目](#旗舰项目--flagship-projects) | [参与共建](#参与共建--join-us) |

> **我们的目标：让 AI 从临时使用的工具，变成个人与小型组织可积累、可审计、可协作的生产能力。**

## 我们如何理解“AI 原生组织”

AI 原生不是给现有流程加一个聊天窗口，而是重新设计工作的基本单元：

```text
人的判断与责任
      ↓
可复用的知识、规则与上下文
      ↓
Agent / Skill / 自动化工作流
      ↓
可验证、可回退、可持续改进的交付
```

因此，OPCspace 关注的不只是工具推荐，还包括：

- 如何把隐性经验沉淀为组织知识；
- 如何让 Agent 理解角色、流程和验收标准；
- 如何在自动化中保留权限、审计和人工责任；
- 如何让一个人或小团队拥有过去只有大型组织才具备的执行系统。

## 参与共建 / Join Us

你可以从一个很小的动作开始：

1. 从带有 [`good first issue`](https://github.com/search?q=org%3Aopcspace+label%3A%22good+first+issue%22+is%3Aissue+is%3Aopen&type=issues) 标签的小任务开始；
2. 通过 Issue 报告过期链接、文档错误或执行失败；
3. 使用“真实案例”模板分享可验证的输入、过程与结果；
4. Fork 旗舰项目并提交 Pull Request；
5. 在组织 Discussions 中提出跨项目问题和社区建议。

外部贡献重点包括文档修正、官方来源更新、真实案例、模板与安全改进。品牌定位、商业产品、合作承诺和发布节奏由 OPCspace 维护团队负责。详细边界见 [贡献指南](https://github.com/opcspace/.github/blob/main/CONTRIBUTING.md)。
