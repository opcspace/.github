# OPCspace 贡献指南

感谢你帮助 OPCspace 从“内容发布”成长为可共同维护的实践系统。

## 优先接受的贡献

- 修正文档错误、失效链接和过期产品信息；
- 补充来自官方来源的事实与核查日期；
- 提交经过脱敏的真实使用案例、模板和验收方法；
- 改善 Agent Skill 的可执行性、安全边界和失败处理；
- 补充测试、构建检查、可访问性和国际化改进。

## 由维护团队负责的内容

- OPCspace 品牌定位、商标和视觉资产；
- 商业产品、定价、课程、训练营和合作承诺；
- 官方政策台账、个人数据和未公开合作资料；
- Release 时间、支持范围和仓库分类；
- 对外代表 OPCspace 的声明。

这些内容可以通过 Issue 提建议，但未经维护者确认不应直接作为既定事实修改。

## 推荐流程

1. 小型文档修正可以直接提交 Pull Request。
2. 新功能、较大结构调整或新 Skill 请先创建 Issue。
3. 从 `good first issue`、`documentation` 或 `case-study` 标签开始最容易被合并。
4. PR 只解决一个清楚的问题，并说明验证方式。
5. 案例必须删除客户姓名、密钥、联系方式和其他敏感信息。

## 内容标准

- 区分官方事实、实践建议和个人推断；
- 易变化信息注明来源与核查日期；
- 不承诺收益、政策资格或产品未公开能力；
- 引用第三方内容时遵守其许可证并保留署名；
- 提交前运行仓库 README 中列出的检查。

参与本组织项目即表示你同意遵守 [行为准则](CODE_OF_CONDUCT.md) 和各仓库许可证。
